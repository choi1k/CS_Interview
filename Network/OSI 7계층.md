# OSI 7계층

7계층을 활용하면 통신 과정을 단계별로 파악할 수 있으며, 문제 발생 시 진단과 대응이 용이함

### OSI 7계층

1. 물리 계층(Physical Layer)

    > 리피터, 케이블, 허브
    
    데이터를 전기적 신호로 변환해 주고받는 기능을 수행

2. 데이터 링크 계층(Data Link Layer)

   > 브릿지, 스위치
   
    물리 계층으로 송수신되는 정보를 관리해 안전한 전달이 가능하도록 함

    Mac 주소를 통해 통신하며, 프레임에 Mac 주소를 부여하고 에러 검출, 재전송, 흐름제어를 진행

3. 네트워크 계층(Network Layer)

    > 라우터, IP
    데이터를 목적지까지 최적의 경로를 활용해 안전하고 빠르게 전달함

    라우터를 통해 이동할 경로를 선택해 IP 주소를 지정하고 패킷을 전달함

    라우팅, 흐름 제어, 오류 제어, 세그먼테이션 진행

4. 전송 계층(Tranport Layer)

    > TCP, UDP
    
    TCP와 UDP 프로토콜을 통해 통신을 활성화함

    * TCP : 전송을 제어하는 프로토콜로 연결 지향 방식이며 높은 신뢰성을 보장함
    
        흐름제어, 혼잡 제어를 제공하며 UDP보다 속도가 느림

    * UDP : 사용자 데이터그램 프로토콜로 비연결 지향 방식이며 신뢰성이 낮음

        흐름제어가 없고 헤더에 존재하는 CheckSum 필드를 통해 최소한의 오류만을 검출하지만 TCP보다 속도가 빠르고 스트리밍과 같은 연속성을 중시하는 작업에 사용됨

5. 세션 계층(Session Layer)

    > API, Socket
    
    데이터가 통신하기 위한 논리적 연결을 담당

    TCP/IP 세션을 생성하고 삭제하는 역할을 함

6. 표현 계층(Presentation Layer)

    > JPEG, MPEG
    
    데이터 표현에 대한 독립성을 제공하고 암호화하는 역할을 수행

    파일 인코딩, 명령어 포장, 압축, 암호화 진행

7. 응용 계층(Application Layer)

    > HTTP, FTP, DNS
    
    최종 목적지로서 응용 프로세스와 직접 관계하여 응용 서비스를 수행함

    사용자 인터페이스, 전자 우편, 데이터베이스 관리 등을 진행
